### Migration tips

- Exclude `node_modules`, `.venv`, `dist`, `build`, `.next`, `.turbo`, and other build caches when rsyncing.
- Run `git status` in each repo before moving; push or stash anything you need to keep.
- After clone, reinstall deps (`pnpm i`, `pip install -r requirements.txt`, etc.).

## Shell and Dotfiles

### Zsh (primary shell)

- `~/.zshrc` — PATH, fnm, pnpm, aliases, Tailscale/SSH shortcuts, starship prompt, Android SDK paths
- `~/.zprofile` — OrbStack shell init

### Bash (secondary / conda)

- `~/.bashrc`
- `~/.bash_profile` — conda init block

### Git

- `~/.gitconfig` — name, email, SSH commit signing via 1Password, aliases, LFS
- `~/.gitignore_global` — DS_Store ignores

### Package managers

- `~/.npmrc` — author defaults and npm registry auth token
- `~/.yarnrc` — yarn metadata (autogenerated)
- `~/Library/pnpm` — pnpm global store path referenced from `.zshrc`

### Misc dotfiles

- `~/.profile`
- `~/.aicommits`
- `~/.condarc`

### Dotfiles repo

- `~/.dotfiles` — Nix/WSL configuration; optional on macOS unless you use Nix here

## SSH and Keys

- `~/.ssh/` — ed25519 and hardware-backed keys, Pi-hole key, `known_hosts`

Transfer securely (AirDrop, encrypted drive, or 1Password/SSH agent export). Re-approve keys in GitHub and on home servers after migration. Update `~/.ssh/config` if you maintain one separately.

## Secrets and credentials

These files contain tokens or passwords. Migrate them manually; never commit contents to this repo.

- `~/.zshrc` — includes home-automation env vars
- `~/.npmrc` — npm registry auth token
- `~/.ssh/*` — private keys
- API keys in app config under `~/Library/Application Support/` or `~/.config/`

Prefer 1Password for secrets on the new Mac where possible.

## App data and accounts

Export or sign in again after apps are installed (see `manual-apps.md`):

- Obsidian vaults

- Figma, Postman, and other logged-in dev tools

## Safe to skip or regenerate

- `~/.cache/`
- `~/.Trash/`
- `node_modules/` inside project dirs
- Homebrew audit snapshots already in this repo (`Brewfile`, `audit/`)
- Duplicate brew list text files in `~/proj`
